4 השוואה: התנהגות צפויה מול בפועל

✅ 11 תווים — התנהגות תקינה
  1. משתמש מקליד תו אסור (כגון ~)
  2. REGEX מחזיר false
  3. מוצגת: "סיסמה לא חוקית"
  4. משתמש מבין מה הבעיה ומתקן
⚠️ 2 תווים — התנהגות בעייתית
  1. משתמש מקליד ' או -
  2. REGEX מחזיר false
  3. אין הודעת שגיאה
  4. משתמש לוחץ "שמור" — הפעולה נכשלת ללא הסבר
הבדל מעניין — מדוע דווקא ' ו- - ? שני התווים האלה שימושיים מאוד בסיסמאות יומיומיות: - (מינוס) — נפוץ מאוד בסיסמאות כגון Pass-2024, ' (גרש) — נפוץ בשמות כגון O'Brien1!. סביר שמשתמשים רבים ינסו דווקא תווים אלה.

6 השפעה על חוויית המשתמש

תרחיש: משתמש מנסה להגדיר סיסמא כגון MyPass-2024! או It'sSecure1!

  • הסיסמא נדחית על ידי ה-REGEX
  • לא מוצגת כל הודעת שגיאה ספציפית
  • כפתור "שמור" נשאר מושבת ללא הסבר
  • המשתמש אינו יודע מה הבעיה ועלול לנסות שוב ושוב
השפעה פרקטית מינוס (-) הוא אחד התווים הנפוצים ביותר בסיסמאות. משתמשים רבים שינסו ליצור סיסמא עם מינוס יתקלו בכשל שקט — ללא כל הסבר מדוע הסיסמא שלהם לא מתקבלת.

8 המלצה לתיקון

יש לוודא שה-validation handler בצד הלקוח מפעיל הודעת שגיאה עבור כל 13 התווים האסורים, ללא יוצא מן הכלל.

בעיה אפשרית בקוד ה-validation message מופעל רק כאשר REGEX מחזיר false וגם התו נמצא ב-character class של הודעת השגיאה — ו-' ו-- חסרים מהרשימה הזו.
פתרון מומלץ הפרד את לוגיקת ה-validation message מה-REGEX הראשי: אם REGEX.test(pw) === false → הצג תמיד "סיסמה לא חוקית", ללא תלות בתו הספציפי.
