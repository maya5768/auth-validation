1 מה בודקים ולמה

כלל 5 ב-REGEX של אימות הסיסמא הוא Negative Lookahead:

(?!.*[ ~`':^()_+\-=?]) ↑ אם כל תו מהרשימה מופיע בסיסמא — ה-REGEX מחזיר false
מה זה Negative Lookahead? (?!.*X) — מסתכל קדימה מבלי לצרוך תווים. אם הוא מוצא את X (כל תו מהרשימה האסורה) בכל מקום במחרוזת — הוא כושל מיד ומחזיר false.

הסקריפט בדף זה בודק רק את כלל 5 — ומוודא שכל אחד מ-13 התווים האסורים גורם לכישלון, ללא קשר למיקומו בסיסמא.

שים לב — 13 תווים, לא 11 הספירה המדויקת מה-REGEX: רווח, ~, `, ', :, ^, (, ), _, +, -, =, ? — סה"כ 13 תווים אסורים.

3 בסיס הסיסמא לבדיקה

כדי לבדוק רק את כלל 5, כל שאר הכללים חייבים להתקיים. הסיסמא הבסיסית:

Abc1!Defgh // 10 תווים תקינים // כלל 1 — אות גדולה: A, D ✅ // כלל 2 — אות קטנה: b, c, e, f, g, h ✅ // כלל 3 — ספרה: 1 ✅ // כלל 4 — תו מיוחד מותר: ! ✅ // כלל 5 — אין תו אסור ✅ // אורך: 10 תווים ✅

לכל מקרי הבדיקה מוסיפים תו אסור אחד במיקום אחד מתוך שלושה:

START תו אסור + Abc1!Defgh
MID Abc1!D + תו אסור + efgh
END Abc1!Defgh + תו אסור
מה צפוי בכל המקרים? כולם אמורים להחזיר false — כלומר הסיסמא נדחית. אם הסקריפט מחזיר true לאחד מהם — זה באג ב-REGEX.
